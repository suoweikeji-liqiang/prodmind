## Part 1：产品认知演进图谱

### Stage 1：工具发散期
- **初始定位**：AI对话工具，帮助非专业PM理清思路
- **核心诉求**：多角色、可配置模型、互相驳斥、内置产品原则

### Stage 2：机制聚焦期
- **认知修正**：从"功能堆砌"转向"冲突引擎+证伪机制"
- **核心价值重定义**：不是功能多，而是强制对抗+强制收敛

### Stage 3：极简收敛期
- **大量砍功能**：
  - 角色市场：P0 → P2
  - 完整PRD：弱化为假设清单
  - 可视化画布：P0 → P1
  - 实时搜索：P0 → P1
- **产出物收敛**：从完整文档转向可验证假设+下一步行动

### 被推翻的观点
- ❌ 角色数量越多越好 → 固定4-5个核心角色
- ❌ 需要完整PRD生成 → 只生成假设清单
- ❌ 需要思维导图导出 → 用Markdown替代
- ❌ 需要角色市场 → 延后到P2

### 反复出现的原则（高频稳定）
1. **证伪优先**（出现 15+ 次）
2. **问题先于方案**（出现 12+ 次）
3. **强制对抗/驳斥机制**（出现 20+ 次）
4. **MVP暴力切割**（出现 10+ 次）
5. **资产强制沉淀**（出现 10+ 次）
6. **场景锚定**（出现 8+ 次）

## Part 2：共识核心（高频稳定原则）

### 底层哲学原则
1. **证伪优先**：AI首要任务是证明你错了，而非附和
2. **问题先于方案**：不定义清楚问题，无法进入方案讨论
3. **验证优于争论**：分歧只能通过实验解决，不能靠辩论
4. **场景锚定**：严禁脱离"用户-任务-环境"的抽象讨论
5. **颗粒度控制**：问题→假设→实验→功能（强制顺序）
6. **可证伪性**：每个结论必须附带"如何证明我错了"

### 核心机制设计

#### 1. 三层架构
- **角色层**：多Agent + 混合模型路由
- **冲突引擎**：强制引用、驳斥检测、冲突高亮
- **资产层**：假设清单、冲突记录、下一步行动

#### 2. 强制辩论流程（不可跳过）
```
用户输入 → 架构师定义问题 → 刺客攻击
→ 用户鬼质疑 → 落地者收敛
```

#### 3. 盲点爆破（系统自动触发）
- 连续3轮无用户视角 → 自动插入用户鬼
- 讨论技术超过5轮 → 刺客强制打断
- 所有人都同意 → 刺客自动反对
- 超过10分钟无结论 → 落地者强制收敛

### 不可删除的功能/结构
1. **固定4角色**：架构师、刺客/魔鬼代言人、用户鬼、落地者/MVP经理
2. **强制产出物**：假设清单、冲突记录、下一步行动（每轮必须生成）
3. **引用反驳机制**：角色必须引用对方观点，不能自说自话
4. **48小时原则**：超时未收敛自动归档

## Part 3：关键冲突清单（必须保留冲突）

### 冲突1：约束强度（核心哲学分歧）
- **分歧主题**：工具应该"强迫"用户还是"引导"用户？
- **立场A**：硬核模式 - 用户不能跳过环节，强制回应质疑
- **立场B**：协作模式 - 用户可随时插入指令，建议进入辩论
- **立场C**：灵活配置 - 默认协作，可开启硬核教练模式
- **根本变量**：用户自律性、产品性格定位、市场接受度

### 冲突2：竞品搜索时机
- **分歧主题**：实时联网搜索是否进入MVP？
- **立场A**：尽快做，是核心差异化
- **立场B**：砍掉，避免复杂度、成本、幻觉
- **根本变量**：成本控制 vs 功能完整性

### 冲突3：动态画布必要性
- **分歧主题**：是否需要可拖拽的可视化画布？
- **立场A**：核心差异化，"不是聊天框"
- **立场B**：用Markdown表格替代，可导入其他工具
- **立场C**：P1做侧边栏预览，真正画布延后
- **根本变量**：开发成本 vs 用户体验差异化

### 冲突4：角色引用机制
- **分歧主题**：角色间必须互相引用还是可选？
- **立场A**：必须互相引用，否则是摆设
- **立场B**：轮询发言+可选驳斥
- **根本变量**：机制强度 vs 实现复杂度

### 冲突5：输出格式（已收敛）
- **分歧主题**：生成完整PRD还是只生成假设清单？
- **初期立场**：完整PRD草稿
- **后期共识**：只生成假设清单+下一步
- **结果**：✅ 已收敛到假设清单

## Part 4：冗余与重复模块

### 重复表达的思想（本质相同）
1. "多角色对抗" = "冲突演化" = "辩论引擎" = "对抗性演化"
2. "盲点扫描" = "盲点雷达" = "盲点爆破" = "完整性检查"
3. "资产沉淀" = "产物输出" = "结构化产出" = "强制沉淀"
4. "MVP经理" = "落地者" = "交付经理" = "MVP Driver"
5. "魔鬼代言人" = "刺客" = "杠精" = "批判官" = "怀疑者"
6. "用户代言人" = "用户鬼" = "Naive User" = "Persona"

### 本质一致但说法不同的部分
- "证伪优先" ≈ "反证思维" ≈ "Disconfirm Bias"
- "场景锚定" ≈ "用户情境至上" ≈ "JTBD框架"
- "强制收敛" ≈ "暴力切割" ≈ "MVP优先"
- "问题先于方案" ≈ "第一性原理" ≈ "先定义问题再设计方案"

### 归并建议
**角色命名统一**：
- 刺客/魔鬼代言人 → 统一为 "刺客"
- 落地者/MVP经理 → 统一为 "落地者"
- 用户鬼/用户代言人 → 统一为 "用户鬼"

## Part 5：结构级提炼版骨架（15条）

### 层级1：产品本质（1条）
**1. 核心定义**
强制对抗式AI辩论工作台，24小时内将模糊想法转化为可验证假设与MVP边界

### 层级2：底层哲学（3条铁律）
**2. 证伪优先**
AI默认立场是"你错了"，直到被证明有价值

**3. 问题锁定**
不定义清楚问题，系统拒绝进入方案讨论

**4. 验证优于争论**
分歧必须落到可测试实验，不能靠辩论解决

### 层级3：系统架构（3层）
**5. 角色层**
固定4角色（架构师、刺客、用户鬼、落地者），可配置不同模型

**6. 冲突引擎**
强制引用反驳机制，角色必须@对方观点进行攻击

**7. 资产层**
每轮强制产出假设清单、冲突记录、下一步行动

### 层级4：核心机制（4个强制）
**8. 强制辩论流程**
用户输入→架构师定义→刺客攻击→用户鬼质疑→落地者收敛（不可跳过）

**9. 强制引用规则**
角色发言必须以"@角色名 针对[观点]"开头，否则系统拒绝

**10. 强制盲点触发**
连续3轮无用户视角/所有人同意/讨论超时，系统自动干预

**11. 强制资产沉淀**
每轮必须生成3项产出（假设/冲突/行动），不可关闭

### 层级5：约束边界（4个不做）
**12. MVP不做联网搜索**
避免成本、幻觉、合规风险（P1可插拔）

**13. MVP不做完整PRD**
只生成假设清单+下一步，不生成完整文档

**14. MVP不做可视化画布**
用Markdown侧边栏预览替代（P1再做）

**15. MVP不做角色市场**
固定4角色跑通机制，再考虑扩展（P2）

## 关键发现

### 核心护城河识别
不是角色数量、不是UI炫酷、不是功能完整，而是：

1. **强制对抗机制**（角色必须互相引用攻击）
2. **证伪优先哲学**（系统默认你错了）
3. **资产强制沉淀**（拒绝聊完即焚）

### 最大未决问题
**约束强度选择**：做"教练"（强迫你变强）还是"顾问"（建议你参考）？

这决定了：
- 产品性格
- 用户体验
- 市场定位

需要通过原型验证用户反应。
