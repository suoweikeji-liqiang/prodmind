# ProdMind 2.0 Web 使用文档

ProdMind 2.0 Web 是一个全栈决策操作系统，通过 6 个 AI 智能体的结构化辩论，帮助产品决策者系统性地验证和压力测试产品想法。

## 快速开始

### 环境要求

- Node.js 18+
- 一个 Supabase 项目（免费版即可）
- 至少一个 AI Provider 的 API Key（OpenAI / DeepSeek 等 OpenAI SDK 兼容服务）

### 安装与启动

```bash
cd prodmind2-web
npm install

# 复制并编辑环境变量
cp .env.local.example .env.local
```

在 `.env.local` 中填入你的 Supabase 凭据：

```
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key
```

启动开发服务器：

```bash
npm run dev
```

访问 `http://localhost:3000`，注册账号后进入「设置」页面配置 AI Provider。

### 首次使用三步走

1. **注册登录** — 使用邮箱注册账号
2. **配置 Provider** — 在「设置」页面添加至少一个 AI Provider（如 OpenAI 或 DeepSeek）
3. **创建会话** — 在首页输入你的决策问题，点击「开始」进入作战室

## 核心概念

### 认知对抗

ProdMind 的核心理念是**通过对抗来验证决策**。系统不会告诉你"这个想法好不好"，而是让 6 个具有不同视角的 AI 智能体对你的想法进行系统性的攻击、质疑和综合分析，帮助你发现盲点。

### 假设驱动

每个产品决策背后都有一组隐含假设。ProdMind 会自动提取这些假设，并追踪它们的验证状态：

| 状态 | 含义 |
|------|------|
| 未验证 (unvalidated) | 刚被提取，尚未经过检验 |
| 已验证 (validated) | 经过辩论和分析，被认为成立 |
| 已否决 (rejected) | 经过辩论，被认为不成立 |

### 信心指数

信心指数（0-100%）是对当前决策可靠性的量化评估，基于：

- **假设验证率** — 已验证假设占总假设的比例
- **风险暴露度** — 高概率 + 高严重性风险的数量（每个高风险项降低 20%）

计算公式：`信心指数 = 假设验证率 × 风险暴露系数 × 100`

### 快照与版本

系统在每轮辩论结束后自动创建快照，记录当时的完整状态（问题定义、假设库、风险库、信心指数）。你可以对比不同版本的快照，观察决策如何随辩论演进。

## 界面导览

### 会话列表页

登录后的首页，展示：

- **统计概览** — 总会话数、活跃会话数、平均轮数、平均信心指数
- **新建会话** — 输入决策问题，点击「开始」创建新会话
- **历史会话** — 按时间倒序排列，显示标题、状态（进行中/已完成）、轮数和信心指数

顶部导航提供「设置」和「退出」入口。

### 作战室（War Room）

作战室是 ProdMind 的核心界面，采用三栏布局：

```
┌──────────┬──────────────────┬──────────┐
│  状态树   │     对话区域      │  快照栏   │
│          │                  │          │
│ · 问题定义│  智能体发言流      │ · 推演路径│
│ · 假设库  │  （实时流式输出）   │ · 快照时间线│
│ · 风险库  │                  │ · 快照对比│
│ · 因果图  │  ──────────────  │          │
│ · 信心趋势│  用户输入栏        │          │
└──────────┴──────────────────┴──────────┘
```

- **左栏 — 状态树**：实时展示问题定义、假设库、风险库、因果环路图和信心趋势图
- **中栏 — 对话区域**：智能体发言的实时流式输出 + 底部用户输入栏
- **右栏 — 快照栏**：推演路径、快照时间线、快照版本对比

移动端下左右栏默认隐藏，通过顶部「状态树」和「快照」按钮切换显示。

## 使用流程

### 1. 创建会话

在会话列表页输入你的决策问题（最多 30 字），例如：

- "是否应该做一个 AI 代码审查工具"
- "社区团购 vs 即时配送哪个方向更好"
- "要不要放弃 B 端转 C 端"

点击「开始」后自动进入作战室。

### 2. 启动第一轮辩论

进入作战室后，点击右上角的「开始」按钮，系统将：

1. 调度本轮需要发言的智能体（第一轮必定包含问题澄清官和反对者代理）
2. 各智能体依次发言，内容实时流式输出到对话区域
3. 自动从发言中提取假设、风险等结构化数据
4. 更新信心指数
5. 自动创建本轮快照

### 3. 参与辩论

每轮辩论结束后，你可以通过底部输入栏进行以下操作：

- **回应** — 输入你的观点或反驳，然后触发下一轮
- **修改问题定义** — 如果智能体帮你发现了更精准的问题表述
- **驳回结论** — 对某个智能体的结论提出异议并说明理由
- **强制下一轮** — 跳过回应直接进入下一轮
- **结束会话** — 返回会话列表

### 4. 多轮迭代

随着辩论推进，系统会根据当前状态动态调度不同的智能体（详见下节）。建议至少进行 3 轮辩论，以便长期价值官参与评估。

### 5. 导出报告

点击作战室右上角的「导出」按钮，下载完整的 Markdown 格式决策报告，包含问题定义、假设库、风险库、推演路径、全部智能体发言记录和版本历史。

## 6 大智能体

ProdMind 2.0 采用 6 个专业化 AI 智能体，各司其职：

### 问题澄清官 (Problem Architect) 🔵

- **职责**：从用户输入中提炼核心问题定义和期望结果
- **激活条件**：第 1 轮必定激活；或当前无问题定义时激活
- **输出**：结构化的问题描述 + 期望结果 + 初始假设列表

### 批判官 (Critical Examiner) 🟡

- **职责**：审查未经验证的假设，指出逻辑漏洞
- **激活条件**：当超过 50% 的假设处于「未验证」状态时激活
- **输出**：未验证假设的详细质疑和分析

### 系统分析官 (System Mapper) 🔷

- **职责**：绘制系统关系图，揭示假设和风险之间的因果关联
- **激活条件**：当假设数 ≥ 3 或风险数 ≥ 2 时激活
- **输出**：系统关系分析和因果链路描述

### 反对者代理 (Devil's Advocate) 🔴

- **职责**：从反面攻击，提出最尖锐的反对意见和风险
- **激活条件**：**每轮必定激活**
- **特点**：使用更高的温度参数（0.8），产生更具创造性的对抗观点

### 风险官 (Risk Architect) 🟣

- **职责**：聚焦高概率、高严重性风险，提出缓解策略
- **激活条件**：当存在高概率或高严重性风险时激活
- **输出**：风险评估（概率/严重性）和应对建议

### 长期价值官 (Strategic Evaluator) 🟢

- **职责**：评估决策的长期战略价值和可持续性
- **激活条件**：从第 3 轮开始激活
- **输出**：长期价值分析和战略建议

### 智能体调度逻辑

系统根据当前会话状态动态决定每轮激活哪些智能体：

| 轮次 | 典型激活组合 |
|------|-------------|
| 第 1 轮 | 问题澄清官 + 反对者代理 |
| 第 2 轮 | 批判官（如假设多数未验证）+ 系统分析官（如假设≥3）+ 反对者代理 + 风险官（如有高风险）|
| 第 3 轮+ | 上述规则 + 长期价值官 |

这意味着辩论越深入，参与的智能体越多，分析越全面。

## 作战室详解

### 左栏：状态树

状态树是决策状态的实时可视化面板，包含以下模块：

**问题定义** — 显示当前版本的问题描述和期望结果。问题定义支持版本化，每次修改都会创建新版本。

**假设库** — 列出所有已提取的假设及其状态（未验证/已验证/已否决）。假设来源于问题澄清官和批判官的分析输出。

**风险库** — 列出所有已识别的风险，标注概率（高/中/低）和严重性（高/中/低）。风险来源于反对者代理和风险官的分析。

**因果环路图** — 以放射状布局展示假设与风险之间的关联关系：
- 内环：假设节点
- 外环：风险节点
- 连线：因果关联

**信心趋势图** — SVG 折线图，展示信心指数随辩论轮次的变化趋势。

### 中栏：对话区域

对话区域是辩论的主战场：

- 智能体发言以颜色编码的卡片形式展示，每个智能体有独特的配色方案
- 发言内容实时流式输出，无需等待完整响应
- 对话按轮次组织，自动滚动到最新消息

**用户输入栏**位于底部，提供以下操作：

| 操作 | 说明 |
|------|------|
| 回应 | 输入你的观点或反驳，触发下一轮辩论 |
| 修改问题 | 重新定义问题描述和期望结果 |
| 驳回结论 | 选择某个智能体，说明驳回理由 |
| 强制下一轮 | 不做回应直接进入下一轮 |
| 结束会话 | 返回会话列表页 |

### 右栏：快照栏

**推演路径** — 基于当前状态探索不同的决策路径。每条路径包含步骤序列和预期结果，帮助你思考"如果走这条路会怎样"。

**快照时间线** — 按版本号排列的快照列表，每个快照记录了创建时的完整状态。支持添加人工判断备注。

**快照对比** — 选择两个快照版本进行结构化对比，直观展示假设、风险和信心指数的变化差异。

## 设置与配置

### AI Provider 配置

在「设置」页面（`/settings`）管理你的 AI Provider：

**添加 Provider**

填写以下信息：

| 字段 | 说明 | 示例 |
|------|------|------|
| 名称 | Provider 标识名 | `openai`、`deepseek` |
| API Key | 你的 API 密钥 | `sk-...` |
| Base URL | API 地址（可选，留空使用 OpenAI 默认） | `https://api.deepseek.com` |
| 默认模型 | 该 Provider 的默认模型 | `gpt-4o`、`deepseek-chat` |

支持所有 OpenAI SDK 兼容的服务，包括但不限于：

- OpenAI（`https://api.openai.com/v1`）
- DeepSeek（`https://api.deepseek.com`）
- 其他兼容 OpenAI API 格式的服务

**角色路由**

你可以为每个智能体单独指定 Provider 和模型，实现混合调用。例如：

- 问题澄清官 → OpenAI GPT-4o（需要精准理解）
- 反对者代理 → DeepSeek（需要创造性对抗）
- 其他角色 → 使用默认 Provider

留空则使用默认 Provider 的默认模型。

### Supabase 配置

详细的 Supabase 配置步骤请参考 [DEPLOY.md](./DEPLOY.md)。关键步骤：

1. 创建 Supabase 项目，获取 Project URL 和 anon key
2. 在 SQL Editor 中执行 `supabase/migrations/001_initial_schema.sql`
3. 为 `sessions`、`assumptions`、`risks`、`agent_comments`、`snapshots` 表开启 Realtime
4. 在 Authentication 设置中启用 Email 登录

## 导出与报告

点击作战室右上角的「导出」按钮，系统会生成一份完整的 Markdown 决策报告。

报告包含以下章节：

- **会话元信息** — 会话 ID、创建时间、总轮数、最终信心指数
- **问题定义** — 最新版本的问题描述和期望结果
- **假设库** — 所有假设及其验证状态（✅ 已验证 / ❌ 已否决 / ❓ 未验证）
- **风险库** — 所有风险及其概率/严重性评级
- **推演路径** — 各推演路径的步骤和预期结果
- **智能体发言记录** — 按轮次组织的完整发言历史
- **决策版本历史** — 快照时间线及人工判断备注

导出的报告可直接用于团队分享、存档或进一步分析。

## 常见问题

### 辩论没有启动 / 智能体不发言

检查以下几点：
1. 是否在「设置」页面配置了至少一个 AI Provider
2. API Key 是否正确且有余额
3. Base URL 是否填写正确（如使用 DeepSeek 需填 `https://api.deepseek.com`）

### 信心指数一直是 0%

信心指数基于假设验证率计算。如果所有假设都处于「未验证」状态，信心指数为 0%。这是正常的——随着辩论深入，假设逐步被验证或否决，信心指数会相应变化。

### 为什么有些智能体没有出现

智能体的激活是动态的，取决于当前会话状态：

- 批判官只在超过 50% 假设未验证时出现
- 系统分析官需要至少 3 个假设或 2 个风险
- 风险官需要存在高概率或高严重性风险
- 长期价值官从第 3 轮才开始参与
- 只有反对者代理每轮必定出现

### 可以同时使用多个 AI Provider 吗

可以。在「设置」页面添加多个 Provider 后，通过「角色路由」为不同智能体分配不同的 Provider 和模型。这样你可以利用不同模型的优势——比如用 GPT-4o 做精准分析，用 DeepSeek 做创造性对抗。

### 移动端如何使用

移动端下作战室的左右栏默认隐藏，通过顶部的「状态树」和「快照」按钮切换显示。点击后对应面板会全屏覆盖，再次点击关闭。

### 数据安全吗

所有数据存储在你自己的 Supabase 项目中，受行级安全策略（RLS）保护：
- 每个用户只能访问自己的数据
- API Key 加密存储，仅用户本人可读取
- 建议在生产环境中启用 SSL 强制连接

### 建议进行几轮辩论

建议至少 3 轮。前 2 轮主要完成问题定义和假设提取，第 3 轮开始长期价值官加入，分析才趋于完整。复杂决策可进行 5 轮以上。
